# 路由器 DDNS 服务功能梳理

## 一、核心功能定位
**目标**：实现类似花生壳的路由器内置DDNS服务，让用户无需额外安装客户端即可实现动态域名解析。

**核心价值**：
- 解决动态公网IP访问问题
- 提供稳定的域名解析服务
- 简化用户配置流程
- 支持主流DNS服务商

## 三、详细功能梳理

### 1. 配置管理模块
- **基本配置**：服务商选择、域名信息、认证信息
- **高级配置**：IP检测频率、检测方式、日志级别、通知设置
- **配置存储**：JSON/XML格式、备份恢复、合法性验证

### 2. IP检测模块
- **检测方式**：公网IP检测、接口IP检测、多源交叉验证
- **逻辑**：IP变化判断（对比上次）、异常处理（重试、冲突）

### 3. DNS更新模块
- **服务商支持**：Kongzhi (本机)、花生壳、No-IP、阿里云、腾讯云、Cloudflare
- **策略**：实时/定时/强制更新
- **流程**：构建请求 -> 发送 -> 处理响应 -> 更新缓存

### 4. 日志管理模块
- **类型**：系统日志、操作日志、错误日志
- **功能**：存储、滚动、查询（时间/级别/关键词）

### 5. 状态监控模块
- **指标**：在线状态、当前IP、上次更新时间、成功率
- **展示**：Web界面、CLI、硬件指示灯
- **告警**：异常/失败告警

### 6. Web管理界面
- **登录认证**：用户/密码、会话管理
- **页面**：配置页面、状态页面、帮助页面

### 7. CLI命令行接口
- **命令**：启动/停止、状态查询、手动更新、配置设置

## 四、实现规划
1. **服务端 (Control Plane)**: 
   - 作为核心 DDNS 服务商 (Provider)。
   - 提供 Web 管理界面 (符合第6点)。
   - 提供标准 API 供路由器调用。
2. **客户端 (Agent)**: 
   - 作为路由器内置服务 (Client)。
   - 实现 IP 检测、DNS 更新逻辑。
   - 提供 CLI 接口。
