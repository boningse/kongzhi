# 产品需求文档 (PRD) - AG网关数据采集系统

## 1. 产品概述
AG网关数据采集系统是一个专为“AG工业物联网智能网关”配套设计的设备配置、数据采集、处理与转发管理平台。
该系统主要用于工业设备数据的采集配置、协议解析控制、数据转发路由、边缘计算以及报警管理。系统通过网关的 V3/V4 报文格式实时获取设备状态，并将历史采集数据存储于 OpenGauss 数据库中，支持工业自动化场景中的海量设备统一管理与分析。

## 2. 用户角色
| 角色 | 核心权限 |
|------|----------|
| 系统管理员 | 管理所有网关设备、网络配置、高级转发规则、平台用户 |
| 运维工程师 | 负责具体网关的点位配置、设备列表维护、边缘计算脚本更新、定时任务管理 |
| 监控用户 | 仅查看“信息总览”、实时数据、历史趋势及报警日志 |

## 3. 核心功能模块

### 3.1 设备连接与认证
- **连接方式**：支持本地局域网连接（IP+密码）和远程连接（网关SN号+密码）。
- **密码管理**：支持“远程管理状态”的开启与关闭（Enable/Disable）。

### 3.2 信息总览 (Dashboard)
- 实时展示网关运行参数：
  - **硬件信息**：网关串号(SN)、固件版本、启动时间、CPU使用率、CPU温度、内存使用量。
  - **业务信息**：网关别名、连接设备数量、采集点位数量、当前网关时间。

### 3.3 网络管理
- **联网方式**：配置网关的 DHCP 或 静态IP (STATIC)。
- **网络参数**：设置网关IP地址、子网掩码、默认网关。修改后支持网关自动重启。

### 3.4 采集管理 (核心配置)
- **设备列表 (通道)**：管理与网关连接的实际设备或下级通道（如某个串口、TCP连接）。
- **点位列表 (数据点/Tag)**：
  - **点位类型**：支持 IO（采集变量） 和 Memory（计算变量）。
  - **上传策略**：周期上传、变化上传、立即上传、不上传。
  - **批量操作**：支持点位模板导出与点位信息批量导入。

### 3.5 转发管理
- 支持配置数据转发的目标和协议：
  - MQTT数据转发（支持写入设备）、HTTP数据转发、Modbus、IEC104及数据库转发。
  - 支持多通道转发的新建、编辑、启停。

### 3.6 边缘计算
- 提供边缘侧计算脚本管理。
- **操作**：新建脚本、编辑脚本、删除脚本。
- **同步**：上载配置（读取网关）、下载配置（写入网关）。

### 3.7 定时任务
- **任务触发类型**：
  - **指定时间**：单次执行。
  - **间隔执行**：按月、日、时、分的周期性重复任务（支持离散值和通配符）。
  - **自定义Cron**：支持6-7位标准Cron表达式（含高级特性如 L、W、#）。
- **执行动作**：设定点位数值（支持Int、Float、BOOL、String最大256字符）。

### 3.8 报警规则
- **四级报警**：高高、高、低、低低报警。
- **抗干扰机制**：
  - **死区阈值 (Deadband)**：防止数据在阈值附近波动导致频繁报警。
  - **持续稳定时间 (S)**：连续超过阈值X秒才触发报警。
  - **恢复稳定时间 (S)**：连续回落至正常区间Y秒才解除报警。

## 4. 业务流程

### 4.1 数据点配置流程
1. 运维工程师登录系统。
2. 建立通信通道（设备列表）。
3. 选择“采集管理”，手动新增或通过模板批量导入点位信息。
4. 设定每个点位的“上传策略”（如周期或变化上传）。
5. 点击“下载配置”将设置同步至网关。
6. 点击“刷新数据”验证实时数据采集是否成功。

### 4.2 数据流向设计 (基于 V3/V4 报文)
1. AG网关根据采集管理配置从底层PLC/仪表获取数据。
2. 数据经由网关按 V3/V4 结构（包含 time, sncode, dev_id, pn, q, v 等字段）打包。
3. 通过 MQTT 或 HTTP Webhook 转发给本数据采集系统。
4. 后端解析报文，验证 q（数据有效性），将遥测记录存入 **OpenGauss** 数据库，并在触发阈值时生成报警记录。

## 5. UI/UX 交互规范
- **布局设计**：左侧为模块导航菜单，右侧为详情与操作区。
- **实时性要求**：信息总览和报警日志需提供自动刷新机制。
- **容错设计**：修改网络配置、下发大量点位配置时，提供确认弹窗及明确的成功/失败反馈。对于复杂定时（如Cron），提供表达式验证及下5次执行时间预览。
